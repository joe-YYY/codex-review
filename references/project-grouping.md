# 项目归并规则

Codex Review 不使用维护者个人项目名作为默认规则。项目归并按以下证据从强到弱判断：

1. 用户在 `.codex-review.json` 中配置的项目规则。
2. 会话工作目录中的明确项目目录名。
3. 用户消息里明确提到的产物目录或文件路径。
4. 与会话真实活动时间接近的近期产物。
5. 会话标题。
6. 通用任务类型，例如“产品与设计”或“系统自动化”。

同一个产物只会归属一个项目。没有足够证据时，报告使用中性分类，不编造项目名称。

## 自定义规则

在要扫描的工作区根目录创建 `.codex-review.json`：

```json
{
  "projectRules": [
    {
      "name": "客户工作台",
      "category": "产品与设计",
      "patterns": ["customer-console", "客户工作台", "workspace/prd"]
    }
  ]
}
```

`patterns` 可以填写目录名、仓库名或稳定关键词。不要填写密钥、Token、完整私人对话或只会出现一次的临时句子。

## 任务类型

当前支持以下通用类型：

- 代码开发
- 文档与内容
- 产品与设计
- 数据与表格
- 研究与分析
- 系统自动化
- Skill 与工具
- 视觉素材
- 其他任务

任务类型用于展示投入分布，不用于替代明确的项目名。
